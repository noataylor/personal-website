Software Series Emacs

Text editing

As part of a recent movement towards open source methodologies in my practice I have begun using Emacs, 'the fully customizable, extensible, self-documenting text editor', a great deal. Whilst I have very little expierence writing Lisp code, I have found Emacs' flexibility as a piece of software quite essential to developing various written documents in a variety of languages. In the image below I am running two Emacs clients in text termials. The clients are clients for an Emacs daemon which is running in the background and I could connect more clients if desired. The displayed buffers show various files and directories relevant to this website. The active buffers synchronise with one another accross clients. In this sense one could regard them as 'instances', or perhaps 'prongs', of the daemon.

Emacs ships with various packages which allow users to send emails, browse the web, manage filesystems, read the news, play games, chat with people online, edit a calendar and so on. Because Emacs is a lisp interpreter, anyone can write a lisp package for a specific purpose, install it on Emacs and use the package, or share the package with others. 'Self-documenting' insofar the user has the possiblity to check, for example, what specfic key-bindings will do in various contexts (local/global; major/minor modes etc.) and access information to faciltate a process. Moreover, as the software is fully customisable one is always at liberty to modify, add to or otherwise change the standard settings.

Emacs is choice software for writing code and, furthermore, it's free!
